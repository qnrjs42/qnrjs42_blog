## Redux에 대해

redux 코드 중에
...state는 바뀌지 않는 데이터는 참조관계로 남고,
바뀌는 데이터만 새롭게 만들어낸다 그러므로 메모리 성능에 효과적

```javascript
{
  ...state,   // 바뀌지 않는 데이터는 재사용
  name: action.data, // 바뀐 데이터는 새로 만듦
}
```

배포 모드에서는 히스토리 모드가 필요없어서 메모리 정리는 계속 하기 때문에 메모리 문제가 일어나지 않는다
